This demo showcases several methods for changing the object stack order, meaning the order in which they are drawn on the canvas.

Ultimately, the `canvas._objects` array determines the stack order. The following functions all re-arrange that array.

[`bringObjectForward`](/api/classes/canvas/#bringobjectforward) & [`sendObjectBackwards`](/api/classes/canvas/#sendobjectbackwards) both accept an optional `boolean` 2nd argument, `intersecting`, which limits changes to intersecting (overlapping) objects.

By default, when an object is selected, it is temporary drawn above all non-selected objects. Setting [`preserveObjectStacking`](/api/classes/canvas/#preserveobjectstacking) to `true` prevents that.
